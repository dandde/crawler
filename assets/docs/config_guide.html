Introduction

Designed for power and flexibility, our configuration system allows you to define scraping jobs as data. Whether you prefer the brevity of YAML, the structure of JSON, or the type-safety feel of TOML, the engine adapts to you.

Zero Clobber

Synchronized terminal output preserves your data while displaying progress.

Recursive Logic

Nesting-capable selectors allow for highly surgical data extraction.

Smart Inheritance

Extend base templates to minimize boilerplate across multiple domains.

Core Structure

Every configuration file defines a SpiderConfig. Here is how it's mapped across formats.

StructureJSON
{
  "name": "full-example",
  "start_urls": ["https://example.com"],
  "root_selector": { "kind": "Class", "spec": "item" },
  "extraction_rules": {
    "title": { "selector": "h1", "extract": "text" }
  },
  "output": { "type": "json", "path": "out.json" }
}
StructureYAML
name: full-example
start_urls:
  - https://example.com
root_selector: .item
extraction_rules:
  title:
    selector: h1
    extract: text
output:
  type: json
  path: out.json
StructureTOML
name = "full-example"
start_urls = ["https://example.com"]
root_selector = ".item"

[extraction_rules.title]
selector = "h1"
extract = "text"

[output]
type = "json"
path = "out.json"

The Selector System

The unified crawler supports two selector modes: Simple Strings and Recursive Objects.

1. Simple Strings

Standard CSS, XPath, or Regex selectors. Use engine prefixes for clarity.

css:.text

Default standard.

xpath://a/@href

XML Path power.

regex:Price: (\d+)

Pattern matching.

2. Recursive Objects

The ultimate control layer. Use these when you need to combine logic (AND/OR/Child/Descendant) at an architectural level.

Recursive MapJSON
{
  "kind": "Child",
  "spec": {
    "parent": { "kind": "Class", "spec": "tags" },
    "child": { "kind": "Tag", "spec": "a" }
  }
}

Translation Reference

Visual Pattern CSS Syntax Structured Variant (TOML)
Combined Matches div.warning kind="And", spec=[{kind="Tag",spec="div"},{kind="Class",spec="warning"}]
Direct Child .container > p kind="Child", spec={parent={kind="Class",spec="container"},child={kind="Tag",spec="p"}}
Attribute Search a[href^="https"] "css:a[href^='https']" (String Mode Only)
Any Depth body footer kind="Descendant", spec={ancestor={kind="Tag",spec="body"},descendant={kind="Tag",spec="footer"}}
Either/Or h1, h2 kind="Or", spec=[{kind="Tag",spec="h1"},{kind="Tag",spec="h2"}]
Positional li:nth-child(2) "css:li:nth-child(2)"

Sophisticated Selectors

For high-precision data mining, the crawler supports advanced CSS and XPath patterns.

Advanced CSS Patterns

Feature Selector Outcome
Starts-with a[href^="https"] Matches all secure links.
Contains a[href*="logout"] Matches links containing "logout".
Nth Child tr:nth-child(even) Selects all even table rows.
Multiple Attr input[type="text"][required] Matches required text inputs.

Complex XPath Expressions

XPath is ideal for navigating relationships that CSS cannot reach.

Relationship XPath Expression Outcome
Sibling Nav xpath://h2/following-sibling::p[1] The first paragraph after an H2.
Text Search xpath://a[contains(text(), "Price")] Links containing the word "Price".
Parent Nav xpath://span[@class="label"]/parent::div The div container of a specific span.
Conditional xpath://div[count(p) > 5] Divs with more than 5 paragraphs.
